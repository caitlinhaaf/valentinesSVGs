Bitmap Vector
  • Made up of pixels
  • Each pixel appears as a tiny square
  • Each pixel is assigned a colour code
  • Can be edited in Adobe Photoshop, Paint
  • Uses math to draw lines and shapes
  • Can zoom in and enlarge without getting pixelated
  • Can be edited in Adobe Illustrator, or a code editor!

SVG's

  • Stands for "Scalable Vector Graphics"
  • Allows us to add or build our own graphics to pages using code to draw lines and shapes
  • What sort of shape do you think the code below would make? What color is it? How big is it?
<svg width="100" height="100"> <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" /> </svg>

SVG's

  • Every SVG is displayed inside of a viewbox
  • The width and height of the viewbox is defined inside the SVG tag (in pixels)

SVG Paths

  • M - "move to", how you define where you want your path to start; will not draw a line
  • L - "line to", draw a straight line to a point
  • C - "curve to", draw a curved line to; you also have to define you point "anchors"
  • Z - "closepath", creates a line to the first point, closing off your line into a shape
<svg width="100" height="100"> <path d="M0,0 L50,50 C50,100 100,50, 100,100" stroke="green" stroke-width="4" /> </svg>

Curved SVG Paths

Drawing curved SVG lines gets a bit more complicated - you have to include the positions of anchor points which will define the direciton of your curve.

CSS

  • Stands for Cascading Style Sheets
  • Used to add styles to your website (i.e. fonts, colors, layout)
  • Can also be used to add animations
  • Syntax uses a selector (the element you want to style), and sets of properties and values.
p{ color: red; background-color: orange; }

CSS Transforms

Used to scale, move, rotate, or even use 3D effects

#square{ transform: rotate(60deg); } #circle{ transform: scale(2); }

We can combine CSS transformations by having them all on one line:

#triangle{ transform: translate(100px, 200px); }

CSS Transform-Origin

  • You can change the effect of the CSS transformation by changing the transform-origin property

CSS Animations

It is possible to animate between different styles in CSS.

Before adding an animation to an element, we have to define the animation.

@keyframes fly { 0% { transform: translateY(0%); } 50% { transform: translateY(-5%); } 100% { transform: translateY(0%); } }

CSS Animations

When applying an animation to an element, we can define:

  • Name - the name we gave the animation we wrote
  • Duration - how long we want our animation to last

You can also define:

  • Iteration Count - how many times we want to run through our animation
  • Animation Direction - do we want our animation to loop? Do we want to animate something from point A to point B and have the animation end?

#bee { animation: fly 4s infinite alternate; }

CSS Motion Paths

  • We can use SVG paths to animate objects along a line by including them in our CSS as an offset-path
  • You can draw an offset path using the same methods we use to draw SVG paths
  • You change the position of the object along the offset-path by setting the offset-distance
#bee { offset-path: path('M20,20 C20,100 200,0 200,100'); animation: fly 3000ms ease-in-out; } @keyframes fly { 0% { offset-distance: 0%; } 100% { offset-distance: 100%; } }
